## 📌 Philosophy

This repo is **not** a solution dump.

- Problem understanding in plain English
- Intuition — how to *think* about the problem
- Brute Force → Optimal progression
- Technique used + **why** that technique
- Dry run with actual values
- Edge cases
- Complexity analysis

## 🏷️ Technique Tags Used

| Tag | Meaning |
|-----|---------|
| `two-pointers/opposite-ends` | L and R from both ends |
| `two-pointers/slow-fast` | Floyd's / cycle detection |
| `sliding-window/fixed` | Window size k is given |
| `sliding-window/variable` | Window size changes dynamically |
| `binary-search/index` | Search on sorted array |
| `binary-search/answer-space` | Search on possible answer range |
| `hashmap/complement` | Two Sum type lookup |
| `hashmap/frequency` | Character/element count |
| `prefix-sum/1d` | Running sum array |
| `dp/0-1-knapsack` | Take or skip DP |
| `dp/lcs` | Longest common subsequence type |
| `graph/bfs` | Level order / shortest path |
| `graph/dfs` | Connected components / cycle |
| `graph/topo-sort` | Dependency ordering |
| `monotonic-stack` | Next greater/smaller element |
| `backtracking` | Generate all possibilities |
| `greedy` | Local optimal → Global optimal |
